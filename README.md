# 🦁 ことばであそぼ！しりとり

**幼児（4〜6歳）の語彙育成を目的とした、AI搭載しりとりアプリ。**
遊びながら「ことば」を覚え、拡散的思考力（あるヒントからいろいろ思いつく力）を育てます。

## なぜ作るのか（プロダクトの意義）

単なるしりとりの実装ではなく、**「しりとりを世の中の課題に接続する」**という方針で設計した。

事実調査の結果、しりとりには次のエビデンスがあることがわかった：

- しりとりは**ワーキングメモリ・想起力・拡散的思考力・語彙力**を鍛える。WHO
  2019「認知症リスク低減ガイドライン」も認知トレーニングを推奨している。
- 幼児は**4歳8ヶ月〜5歳4ヶ月**でしりとりが可能になり、使う語が「動物→食べ物」と**カテゴリを拡大**しながら語彙を増やす。
- 既存のしりとりアプリは「音声・CPU対戦」に寄っており、**「幼児の語彙育成に特化し、AIで優しく導く」**方向は差別化の余地がある。

そこで本アプリは、**幼児がひとりでも／親と一緒でも、優しく語彙を増やせる知育プロダクト**を目指した。

## 実装機能

### 必須機能

1. **直前の単語を表示**（大きな文字＋よみ＋「つぎは○から！」）
2. **任意の単語を入力**できる
3. **末尾↔先頭の一致判定**：一致で更新／不一致はやさしくエラー表示
4. **「ん」で終わるとゲーム終了**
5. **既出単語の再入力でゲーム終了**
6. **リセット機能**（ゲーム中・終了後どちらからも）

### 追加機能

- **①
  育成の可視化（履歴）**：使った単語を一覧表示し、**カテゴリ別の語彙数**（どうぶつ／たべもの…）を集計。終了時に「きょうは○こ
  つかえたね！」と成果を提示 → 親が子の成長を実感できる。
- **②
  ヒント（段階スキャフォールド）**：しりとりが鍛える「拡散的思考力」を奪わないよう、**答えではなく"思考の足場"を段階的に開示**する。
  1. カテゴリ示唆（「"こ"から始まる、どうぶつにいるかな？」）
  2. 連想なぞなぞ（AI生成：「白くてふわふわ、"こ"から…なーんだ？」）
  3. 頭2文字だけ開示
  4. 最終手段として具体語を提示
- **③ 読み上げ**：Web Speech
  API（`SpeechSynthesis`）で単語・ヒントを音声読み上げ。字が読めない幼児でも遊べる。
- **④ AIによる言葉判定**：入力語の**よみ・実在性・カテゴリ**をAIが判定（下記）。

## 設計思想：AIと決定論の切り分け

しりとりの技術的な難所は「単語のよみ（末尾/先頭のかな）」の取得にある。「林檎」でも「りんご」でも末尾が「ご」だと知るには**読み仮名**が要る。静的辞書は未知語で詰む。

そこで **LLM（Claude Haiku 4.5）に「単語 →
よみ・実在性・カテゴリ・ヒント」を判定させる**ことでこの難所を解いた。ただし——

> **ゲームの勝敗ルール（末尾一致・「ん」終了・既出）は、AIのゆらぎに委ねず、必ず決定論コードで確定させる。AIは"ゆらぎのある判断"（よみ・実在・カテゴリ・ヒント）だけを担う。**

この切り分けにより、ハルシネーションが勝敗を狂わせることを防いでいる。

APIキーはクライアントに晒せないため、**Denoサーバーが安全なプロキシ**としてAIを呼ぶ。これが「なぜサーバーがあるか」の技術的必然になっている。

```
[ブラウザ] ──入力語──▶ [Deno /api/judge] ──▶ [Anthropic API]
  勝敗は決定論で確定      ANTHROPIC_API_KEY を        Claude Haiku 4.5
  (public/shiritori.js)   env から取得（秘匿）    {よみ,実在,カテゴリ}
```

**縮退運転（フォールバック）**：AI障害・APIキー未設定時も、実在チェックをスキップして**末尾一致の基本ルールだけでゲームを続行**する（画面に「AIおやすみちゅう」を表示）。ゲームは止まらない。

## 技術構成

| 項目       | 採用                                               |
| ---------- | -------------------------------------------------- |
| ランタイム | Deno                                               |
| サーバー   | `Deno.serve` ＋ `@std/http` の静的配信             |
| フロント   | 素の HTML / CSS / JavaScript（フレームワークなし） |

```
│   ├── index.html
│   ├── app.js            # ゲーム進行・状態管理・読み上げ
│   ├── shiritori.js      # 決定論ロジック（ブラウザ・Deno 共用）
│   └── style.css
└── deno.json
```

## ローカルで動かす

```bash
# 1. APIキーを設定（AI機能を使う場合）
export ANTHROPIC_API_KEY="sk-ant-..."

# 2. 開発サーバー起動（http://localhost:8000）
deno task dev

# 3. テスト
deno task test
```

APIキーを設定しなくても、縮退運転で基本のしりとりは遊べる。

## デプロイ（Deno Deploy）

1. GitHub のパブリックリポジトリに push する
2. [Deno Deploy](https://deno.com/deploy)
   でプロジェクトを作成し、本リポジトリを連携（エントリポイント：`main.ts`）
3. 環境変数 **`ANTHROPIC_API_KEY`** を Deno Deploy の Secret に設定する
4. デプロイされたURLを本READMEの「デプロイURL」に記入

## AI使用箇所の詳細

本アプリは **Anthropic Claude Haiku 4.5**
を以下の2エンドポイントで使用している（実装は `lib/ai.ts`）。いずれも
`temperature=0` ＋ **tool による構造化出力**で、出力のブレを抑えている。

### `POST /api/judge` — 言葉判定

- **入力**：ユーザーが入力した単語（漢字・カタカナ・ひらがな可）
- **AIの役割**：現代仮名遣いの**よみ**、**実在する言葉か**、**幼児向けカテゴリ**を判定
- **使い道**：よみを決定論ロジック（`public/shiritori.js`）に渡して勝敗を確定。実在しない語はやさしく差し戻し（知育）。カテゴリは履歴集計に使用

### `POST /api/hint` — 段階ヒント生成

- **入力**：次の頭文字・ヒントレベル・使用済み単語・カテゴリ
- **AIの役割**：頭文字から始まる幼児向けの語を選び、**答えを言わない連想なぞなぞ**を生成
- **使い道**：レベルに応じて開示（なぞなぞ→頭2文字→具体語）

### 開発におけるAI活用

本プロジェクトの設計方針の壁打ち・事実調査・実装・コードレビューに、Claude
Code（Anthropic）を利用した。
